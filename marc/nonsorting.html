Introduction

With Proposal No 98-16R (Nonfiling characters in all MARC formats), the MARC 21 community approved the use of beginning and ending control characters to block off nonfiling or non-sorting characters in MARC 21 records. These new MARC 21 characters are based on the NSB and NSE characters of ISO 6630 (Bibliographic Control Numbers). For the MARC-8, UCS/Unicode representations of these characters, see the appropriate MARC 21 Specifications pages.

Discussion Paper 118 and Discussion Paper 2002-DP05 explored specific guidelines on how to apply these characters. After much discussion, it was decided that prescriptive guidelines should be drafted that present using the control character technique to indicate non-sorting characters, particularly for initial articles, in a limited number of fields. These guidelines with the allowed fields and subfields are presented below.

The application of this new technique will be limited to identification of nonfiling initial characters in access point fields and subfields. It could, however, be potentially used in a future phase of implementation for the identification of interpolated non-sorting characters (i.e., to omit [i.e.], [sic]) or GMDs from the sort form of headings. It is expected that systems will accommodate both the present technique of nonfiling indicators and the new technique using non-sorting control characters, although they must not be used at the same time for a specific non-sorting zone. This technique is not to be used to identify character strings to be excluded from keyword indexing.

In the following examples, the non-sorting control characters are represented as curly braces { } and the blank (Hex 20) indicator value is represented as the pound character #.

Guidelines for the Non-Sorting Control Character Technique in the MARC 21 Formats

1. Non-Sorting Zone

The use of the phrase "non-sorting zone" encompasses one or more non-sorting characters. The following rules are assumed in identifying the extent of a non-sorting zone:

  • Any nonfiling or non-sorting character and any diacritic associated with a non-sorting character is considered part of the non-sorting zone. This includes initial articles and diacritics associated with them.

  • Any space or mark of punctuation preceding the first sorting character is considered part of the non-sorting zone. A romanized alif (ANSEL AE=U+02BE) or romanized ayn (ANSEL B0=U+02BB) preceding the first sorting character is also considered part of the non-sorting zone.

2. Using Non-Sorting Control Characters in the MARC 21 Formats

Non-sorting control characters may be used in the MARC 21 fields and subfields identified in the following lists. In some cases, non-sorting control characters are used in place of nonfiling indicator values. All tags followed by an asterisk (*) in the following lists currently have nonfiling indicator values defined (see column 4 "Indicator Position").

When the non-sorting control characters are used for a non-sorting zone for which the nonfiling indicator is defined, the nonfiling indicator value must be set to zero. When applied to initial articles, the two techniques are mutually exclusive and use of both for a specific non-sorting zone will result in errors or unpredictable processing of the field.
