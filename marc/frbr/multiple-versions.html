NDMSO has added display examples to accompany the study as Section 4.

1. Introduction

The feasibility of creating records describing multiple manifestations of the same work has been a recurring topic of discussion within the MARC community for more than a decade, but to date there has been no effective resolution of the issue. Within the last few years, the issue has become even more pressing as a result of the increasing use of digital technologies both to create original materials and to convert existing materials to digital form, and the resulting proliferation of alternative manifestations of the same content.

A number of recommendations relating to the "multiple versions" issue were put forward at the Bicentennial Conference on Bibliographic Control in the New Millennium, hosted by the Library of Congress in November 2000. Several of those recommendations, subsequently incorporated into the LC action plan, relate more or less directly to the MARC format. The following are the specific tasks from the action plan:

  • development of functional requirements for systems to manage separate records for related manifestations at the global level and to consolidate them for display at the local level (near-term) (2.4);

  • development of guidelines for determining when a single record rather than separate records should be created for manifestations of an expression (near-term) (3.5);

  • enhancements to the MARC formats to support the display of hierarchical relationships among records for a work, its expressions, and its manifestations, based on FRBR (longer term) (3.4).

Outlined below are a number of conclusions drawn from the mapping of MARC data elements to FRBR that are directly relevant to those three MARC-related recommendations in the LC action plan.

2. Display of Records for Multiple Manifestations of the Same Work

2.1 Hierarchical Level Tables

The tables below illustrate how a hierarchical display of records for multiple manifestations of the same work could be generated from data contained in separate records for the individual manifestations. The hierarchical display comprises five levels of data:

  1. a citation for the work (Table 1)

  2. "guide cards" for the various expressions of the work (Table 2)

  3. brief entries for the various manifestations of each expression - title/edition/numbering (Table 3)

  4. brief entries for the various manifestations of each expression - imprint (Table 4)

  5. brief entries for the various manifestations of each expression - physical description/standard number (Table 5)

The tables were compiled using the mapping of FRBR to MARC data elements in Functional Analysis of the MARC 21 Bibliographic and Holdings Formats, Table 2, Table 3, and the basic data requirements identified in Functional Requirements for Bibliographic Records, Tables 7.3, 7.4, 7.5, and 7.8 (pp. 102-105, 109).

For each of the five levels in the hierarchical display, the tables contain the following:

  • Column 1: identifies the user task supported by that level of the display.

  • Column 2: identifies the attributes and relationships associated with the work, expression, or manifestation that need to be reflected in order to support the designated user task.

  • Column 3: identifies the data elements in the MARC record that would be displayed for each of the designated attributes or relationships.

  • Column 4: identifies the coded data in the MARC record that could be used to verify that all records collocated under the citation for a work or a "guide card" for an expression are relevant to that work or expression.

  • Column 5: provides explanatory comments on the generation and grouping of data for the hierarchical display.

2.2 Matching Data for the Work

For the purposes of this analysis, records for the "same work" are considered to include two groups:

  1. records that match on the name subfields in the main entry (100/110/111) and title (240/243/245 - in that order of precedence - subfields $a, $n, and $p). In cases where main entry is under title (130, or 245 - where there is no 1XX field), records that match on the title subfields are considered to be records for the same work.

  2. records that contain an analytical added entry (700/710/711/730/740, with a second indicator value of 2) that matches the main entry/title elements in records from group one (1) above.

Added entries of an unspecified type (700/710/711/730/740, where the second indicator value is blank) are considered, by default, to be either secondary added entries or related work added entries and are not taken into account in the matching process.

2.3 Display

An assumption is made that the third, fourth, and fifth levels in the hierarchical display (i.e., the brief entries for the manifestations) would be linked to the full records for the manifestations identified, and that any additional data the user required for purposes of selecting or obtaining that manifestation, or for purposes associated with resource use, could be found in the linked record. The linked record would also provide additional information on the work (e.g., intended audience), the expression (e.g., projection, representation of relief), and on relevant work-to-work, expression-to-expression, and manifestation-to-manifestation relationships). Therefore, the data included in the hierarchical display is limited to that which is essential to identify the work, expression, or manifestation, and a limited number of data elements to assist the user in selecting appropriate manifestations among those identified.

For purposes of hierarchical display, the citation for the work would be displayed only once, preceding the first of the "guide cards" for the various expressions of the work. Brief entries for the various manifestations would be listed under the appropriate "guide card". If all the data recorded in the first two segments of the brief entry for the manifestation are applicable to more than one physical form of that manifestation (e.g., in the case of a sound recording released simultaneously on tape and on compact disc) a single brief entry combining title/edition/numbering and imprint could be listed, followed by physical descriptions and standard numbers for each of the formats. Similarly, for manifestations bearing the same title, statement of responsibility, edition statement, and numbering, but issued by different publishers and/or at different times, a single brief entry for title/edition/numbering could be listed, followed by separate listings for the various imprints, each of which in turn would be followed by one or more segments containing the physical description/standard number applicable to each of the imprints.

Use the specifications for hierarchical display set out in Tables 1-5 as input to the development of functional requirements for systems to manage the consolidated display of separate records for related manifestations.

3. Single Record for Manifestations of an Expression

The recommendation in the LC action plan regarding the development of guidelines for determining when a single record rather than separate records should be created for manifestations of an expression is set in the context of AACR, and does not refer explicitly to the MARC format. It is obvious, however, that any changes to current AACR practice with respect to the inclusion of data for multiple manifestations within a single descriptive record could have implications for the MARC format. Conversely, the degree of flexibility that exists within the current MARC format structure to accommodate changes in AACR practice may be a constraining factor in the consideration of options for the proposed AACR guidelines.

As for structural constraints within the MARC format structure that could have a bearing on options under consideration, the following facts need to be highlighted.

3.1 Field Repeatability Concern

Current specifications as to repeatability of variable fields allow only one occurrence of each of the following fields:

  • 245 - Title statement
  • 250 - Edition statement
  • 254 - Musical presentation statement
  • 256 - Computer file characteristics
  • 257 - Country of producing entity for archival films
  • 306 - Playing time
  • 310 - Current publication frequency
  • 507 - Scale note for graphic material
  • 514 - Data quality note

Given the current policy commitment to maintain compatibility between AACR and ISBD(G), it is unlikely that any guidelines developed in response to the action plan's recommendation would involve repeatability of either the title statement or the edition statement. It can probably be assumed, therefore, that any guidelines developed for AACR would limit the single-record description to manifestations bearing the same title, parallel titles, other title information, statements of responsibility, edition statement, statements of responsibility relating to the edition, statement relating to a named revision of an edition, and statements of responsibility relating to a named revision of an edition. Presumably, the general material designation, if not applicable to all the manifestations described in the record, would simply be omitted. The non-repeatability of fields 245 and 250 should therefore not be an issue.

The data recorded in fields 254 and 256 correlate with attributes of expression in FRBR (type of score and extent of expression, respectively). It is reasonable to assume, therefore, that any guidelines developed for AACR would require separate records for manifestations of the same musical work presented in a different form, or for computer files that differed with respect to the number of files, records, statements, etc. If that is not the case, fields 254 and 256 would have to be made repeatable.

As for field 257, practices within the archival film community would have to be examined more closely before determining whether the non-repeatability of the field would pose any problems with respect to using a single record for multiple manifestations of the same motion picture.

As playing time, publication frequency, and scale correlate with attributes of expression in FRBR (extent of expression, frequency, and scale, respectively), it is reasonable to assume that any guidelines developed for AACR would require separate records to be created for manifestations of the same work exhibiting differences in any of those attributes. If that is not the case, fields 306, 310, and 507 would have to be made repeatable.

If the guidelines developed for AACR were to permit a single record describing manifestations that differed in data quality, field 514 would have to be made repeatable.

3.2 Field 008 Interpretation

Fixed-length data elements are a different matter. Current specifications as to the repeatability of fields containing fixed-length data elements permit the use of one or more occurrences of field 006 in conjunction with field 008 only for purposes of recording data that pertains to components or special aspects of the item described in the record. As field 008 itself is not repeatable, consistency with current MARC practice would limit the use of a single record describing multiple manifestations to those instances where the code values recorded for each of the data elements defined in field 008 would have to apply to all manifestations described in the record.

For the most part, the data elements defined in field 008 pertain either to the manifestation or to the expression. Assuming that any guidelines proposed for describing multiple manifestations in a single record would be limited to manifestations of the same expression of the work, those data elements would pose no problem. However, as noted in the mapping of MARC to FRBR (Table 3), the following data elements defined in field 008 correlate with attributes of the manifestation:

  • form of item (form of carrier)
  • form of original (form of carrier for a related manifestation)
  • original alphabet of title (original alphabet of title)

As for form of item, it is highly unlikely that any one of the code values currently defined would apply to all manifestations described in such records. To accommodate records describing multiple manifestations, a new code value would have to be defined for form of item to reflect the fact, where applicable, that a variety of carriers are described in the record. To allow for the specific coding of the various forms of carrier described, the definition and scope of field 007 could be modified to indicate that in records describing more than one manifestation, multiple occurrences of field 007 could be used to code the physical characteristics of the different manifestations. Form of carrier could then be recorded in character position 01 (specific material designation) of a separate occurrence of field 007 for each manifestation.

The fixed-length data element for form of original (defined in field 008 for serials) would not present a problem, provided all manifestations described in the record were related to the same original.

The fixed-length data element for original alphabet of title (defined in field 008 for serials) should not present a problem, provided the guidelines allow the use of a single record only in cases where the title and statement of responsibility area (field 245) is identical for all the manifestations described. If that is the case, the original alphabet of the title could be assumed to be the same for all the manifestations, provided AACR instructions for the transcription of title information are followed.

Recommendation:

Convey the characteristics inherent in the MARC format to those responsible for developing the AACR guidelines for describing multiple manifestations in a single record.
